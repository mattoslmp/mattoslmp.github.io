## Migrar de `mattoslmp.github.io` para `databiomics.github.io`

Não é possível "renomear" diretamente um site de usuário do GitHub Pages (`<usuario>.github.io`) sem trocar o nome de usuário/organização no GitHub.

Opções seguras para não quebrar o site atual:

1. **Manter `mattoslmp.github.io` como está** (já configurado em `url` no Jekyll).
2. **Criar uma organização/usuário `databiomics`** e publicar o mesmo conteúdo em `https://databiomics.github.io`.
3. **No site antigo, manter redirecionamento** para o novo domínio por um período de transição.
4. **Atualizar os links públicos** para o novo endereço apenas após validar tudo.

> Se preferir um domínio próprio (ex.: `databiomics.com`), use arquivo `CNAME` + configuração de DNS no provedor de domínio.
